ISAAC SIM E2E MISSION — MANUAL LAUNCH ORDER
===========================================

Aktualni overeny postup pro Full E2E mission v Isaac Sim.

DULEZITE
--------
- ROS2 E2E scenar spoustej az POTOM, co je Isaac Sim opravdu propojeny s PX4.
- Pokud PX4 stale vypisuje:
  "Waiting for simulator to accept connection on TCP port 4560"
  tak Isaac jeste neni korektne pripojeny.
- Pokud se v Isaac/Pegasus objevi chyba typu:
  "World object invalidated" nebo
  "AttributeError: 'World' object has no attribute '_scene'"
  znamena to, ze vehicle byl loadnuty do neplatneho nebo jeste neinicializovaneho stage/worldu.
- `colcon build` poustej z rootu workspace:
  `/home/tj/_Data/_Projekty/TJlabs/scout_ws`
  Ne z `~`, jinak `colcon` prochazi cizi virtualenvy a vypisuje hromadu
  nesouvisejicich chyb o `Cython`, `numpy.distutils`, `omni` apod.

V ISAAC SIM UDELEJ PRESNE TOHLE
-------------------------------
1. Otevri stage:
   /home/tj/_Data/_Projekty/TJlabs/scout_ws/worlds/agro_field.usd

2. Pockej par sekund, az se stage kompletne nacte.
   Neprovadej hned Load Vehicle.

3. V Pegasus UI vyber PX4 backend.

4. Teprve potom klikni na Load Vehicle.

5. Otevri `Window > Script Editor`.

6. Spust v nem:
   exec(open("/home/tj/_Data/_Projekty/TJlabs/scout_ws/Pegasus_scenarios/simulation_cam.py").read())

7. Zkontroluj v Isaac logu, ze skript nasel kameru a nainstaloval publishery.
   Ocekavany vysledek je publish na:
   - `/drone_0/camera/image_raw`
   - `/drone_0/depth/image_raw`

8. Script pust jen jednou za session.
   Pokud ho pustis opakovane, muzes vytvorit duplikatni publishery.

9. Potom dej Play.

10. Sleduj TERMINAL 1 s PX4.
   Jakmile zmizi:
   "Waiting for simulator to accept connection on TCP port 4560"
   a PX4 zacne normalne komunikovat se simulatorem,
   je Isaac <-> PX4 propojeni hotove.

-----------------------------

Poznamky:

- ten skript se pousti az uvnitr beziciho Isaac Sim ze Script Editoru

Default topicy pro kameru/depth, ktere `gcs_bridge` standardne odebira, jsou:
- `/drone_{index}/camera/image_raw`
- `/drone_{index}/depth/image_raw`

Aktualni overeny single-drone stav po spusteni `simulation_cam.py` v Isaac session:
- `/drone_0/camera/image_raw`
- `/drone_0/depth/image_raw`

`/drone_0/camera/camera_info` se muze objevit podle konkretniho Isaac build/runtime helperu,
ale pro aktualni workflow neni nutny.

RYCHLA DIAGNOSTIKA KDYZ NEFUNGUJE KAMERA
----------------------------------------
1. Over, ze backend launch bezi:
   ros2 node list | grep gcs_bridge

2. Over, jake topicy Isaac publikuje:
   ros2 topic list | grep -E 'image|camera|depth'

3. Ocekavany stav pro aktualni `simulation_cam.py` je:
   /drone_0/camera/image_raw
   /drone_0/depth/image_raw

4. Over, ze na nich opravdu tecou data:
   ros2 topic echo /drone_0/camera/image_raw --once
   ros2 topic echo /drone_0/depth/image_raw --once

5. Over publisher count:
   ros2 topic info /drone_0/camera/image_raw -v
   ros2 topic info /drone_0/depth/image_raw -v

6. Pokud vidis `Publisher count: 2`, skript byl v te same Isaac session spusten
   dvakrat. Pro cisty stav restartuj Isaac a pust `simulation_cam.py` jen jednou.

7. Pokud topic nesouhlasi s defaultem `/drone_0/camera/image_raw` a
   `/drone_0/depth/image_raw`, uprav Pegasus scenar nebo launch parametry
   `camera_topic_template` / `depth_topic_template`

KDYZ TO SPADNE
-------------
Pokud Pegasus/Isaac hlasi chyby jako:
- new stage was opened
- World invalidated
- 'World' object has no attribute '_scene'

tak typicky pomuze:
1. zavrit vehicle load flow
2. znovu otevrit stage
3. pockat na plne nacteni
4. az potom znovu PX4 backend + Load Vehicle + Play

Pokud PX4 zustane viset na:
"Waiting for simulator to accept connection on TCP port 4560"
tak ROS2 cast mise zatim nespoustej.
