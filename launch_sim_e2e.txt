ISAAC SIM E2E MISSION — MANUAL LAUNCH ORDER
===========================================

Aktualni overeny postup pro Full E2E mission v Isaac Sim.

DULEZITE
--------
- ROS2 E2E scenar spoustej az POTOM, co je Isaac Sim opravdu propojeny s PX4.
- Pokud PX4 stale vypisuje:
  "Waiting for simulator to accept connection on TCP port 4560"
  tak Isaac jeste neni korektne pripojeny.
- Pokud se v Isaac/Pegasus objevi chyba typu:
  "World object invalidated" nebo
  "AttributeError: 'World' object has no attribute '_scene'"
  znamena to, ze vehicle byl loadnuty do neplatneho nebo jeste neinicializovaneho stage/worldu.
- `colcon build` poustej z rootu workspace:
  `/home/tj/_Data/_Projekty/TJlabs/scout_ws`
  Ne z `~`, jinak `colcon` prochazi cizi virtualenvy a vypisuje hromadu
  nesouvisejicich chyb o `Cython`, `numpy.distutils`, `omni` apod.

-------------------------------

AZ POTOM SPOUSTET ROS2 CAST MISE
--------------------------------

TERMINAL 5 — ROS2 E2E backend
-----------------------------
cd /home/tj/_Data/_Projekty/TJlabs/scout_ws
source /opt/ros/jazzy/setup.bash
source install/setup.bash
ros2 launch scout_control isaac_e2e_mission.launch.py

Tento launch dnes spusti:
- field_setup_coordinator
- home_manager
- manual_controller (headless backend pro Swarm Center Manual tab)
- swarm_agent_0
- swarm_agent_1
- swarm_coordinator
- cell_data_recorder
- spray_controller
- ml_interface
- mission_launcher
- gcs_bridge

Poznamky:
- `swarm_agent_1` se spusti taky, ale realny vyznam ma az pri `drone_count:=2`
- launch nepouziva `ros_gz_bridge` ani `ros_gz_image`
- kamera v Isaac vetvi jde primo z ROS2 topicu do `gcs_bridge`
-
Default topicy pro kameru/depth, ktere `gcs_bridge` standardne odebira, jsou:
- `/drone_{index}/camera/image_raw`
- `/drone_{index}/depth/image_raw`

Pokud Isaac publikuje kameru jinam, nejdriv si over realny topic:
ros2 topic list | grep -E 'image|camera|depth'

A podle toho pak uprav bridge / relay tak, aby data tekla do:
- `/drone_0/camera/image_raw`
- `/drone_0/depth/image_raw`

cd /home/tj/_Data/_Projekty/TJlabs/scout_ws/swarm_center

CO DELAT DAL
------------
1. Ve Swarm Center otevri tab `Manual`.
2. Fokusuj tab a letej WSAD + sipky nahoru/dolu.
3. Uloz home pad pres `Set pad_0`.
4. Pokud jedes 2 drony, uloz i `Set pad_1`.
5. Oznac 4 rohy pole pres `Mark NE/NW/SE/SW`.
6. Pockej, az `field_setup_coordinator` vygeneruje grid a vrati dron domu.
7. Spust misi tlacitkem `Start Mission`.


RYCHLA DIAGNOSTIKA KDYZ NEFUNGUJE KAMERA
----------------------------------------
1. Over, ze backend launch bezi:
   ros2 node list | grep gcs_bridge

2. Over, jaky topic Isaac publikuje:
   ros2 topic list | grep -E 'image|camera|depth'

3. Over, ze na nem opravdu tecou data:
   ros2 topic echo <camera_topic> --once

4. Pokud topic nesouhlasi s defaultem `/drone_0/camera/image_raw`,
   uprav bridge / relay tak, aby kamera tekla do tohoto topicu


RYCHLA DIAGNOSTIKA KDYZ NEFUNGUJE 3D MAPA POLE
----------------------------------------------
1. Over, ze existuje grid:
   ls /home/tj/_Data/_Projekty/TJlabs/scout_ws/perimeters/field_grid.json

2. Pokud grid jeste nevznikl, 3D outline pole se nema z ceho vykreslit.

3. Pokud 3D tab ukazuje jen fallback text, chybi `pyqtgraph` nebo `PyOpenGL`:
   cd /home/tj/_Data/_Projekty/TJlabs/scout_ws/swarm_center
   pip install -r requirements.txt


KDYZ TO SPADNE
-------------
Pokud Pegasus/Isaac hlasi chyby jako:
- new stage was opened
- World invalidated
- 'World' object has no attribute '_scene'

tak typicky pomuze:
1. zavrit vehicle load flow
2. znovu otevrit stage
3. pockat na plne nacteni
4. az potom znovu PX4 backend + Load Vehicle + Play

Pokud PX4 zustane viset na:
"Waiting for simulator to accept connection on TCP port 4560"
tak ROS2 cast mise zatim nespoustej.
