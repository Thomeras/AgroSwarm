# Scout Swarm Center

Scout Swarm Center is the PyQt6 ground control station for the Scout
Autonomous Swarm project. It provides operator control, mission monitoring,
camera viewing, and 3D situational awareness for the Phase 1-5 E2E milestone.

## Role In The System

Swarm Center is not a PX4 flight controller. It is an operator station that
sends mission/setup intent to the ROS2 backend and displays telemetry from PX4
and ROS2.

Connections:

- MAVLink UDP on ports `14540+` for PX4 SITL telemetry and arm/disarm actions.
- TCP bridge on port `17845` through the ROS2 `gcs_bridge` node.
- Optional grid/report assets generated by the E2E mission backend.

## Architecture

```text
PX4 SITL / Gazebo
  | MAVLink UDP 14540+
  v
Swarm Center
  | TCP JSON bridge 17845
  v
ROS2 scout_control backend
  - gcs_bridge
  - field_setup_coordinator
  - swarm_coordinator
  - swarm_agent_N
  - obstacle_avoidance_runtime_N
```

Important implementation modules:

```text
swarm_center/
|-- main.py                    # application entry point and CLI arguments
|-- core/
|   |-- bridge_protocol.py     # JSON bridge message protocol
|   |-- ros2_bridge.py         # TCP client thread for gcs_bridge
|   |-- mavlink_manager.py     # MAVLink telemetry/control threads
|   |-- swarm_manager.py       # shared mission and drone state
|   |-- field_manager.py       # field/grid loading and coordinate helpers
|   |-- field_model_loader.py  # mapping/model artifact loading
|   `-- report_generator.py    # HTML mission report generation
`-- ui/
    |-- main_window.py         # main PyQt6 window
    |-- field_view.py          # 2D field/grid view
    |-- camera_view.py         # per-drone camera/depth stream view
    |-- viewport_3d.py         # 3D field/drone view
    |-- drone_list.py          # drone telemetry/status table
    |-- control_panel.py       # mission mode and RTH controls
    `-- manual_control.py      # operator setup/manual intent controls
```

## Running

From the repository root:

```bash
python3 swarm_center/main.py --drones 4
```

Optional full form:

```bash
python3 swarm_center/main.py \
  --drones 4 \
  --base-port 14540 \
  --bridge-host 127.0.0.1 \
  --bridge-port 17845 \
  --grid-file perimeters/field_grid.json
```

The recommended public milestone workflow starts Swarm Center through
`scout_launcher.py`, together with PX4 SITL, Gazebo, Micro XRCE-DDS, and the
ROS2 `full_e2e_mission.launch.py` backend.

## Current UI Areas

- Mission view: field grid, drone positions, setup status, mission progress.
- Manual/setup controls: pad marking, boundary capture, grid generation, and
  mission confirmation through ROS2 intent topics.
- Drone list: per-drone telemetry, mission assignment, and arm/disarm actions.
- Camera view: per-drone image/depth streams forwarded through `gcs_bridge`.
- 3D map: drone trails and field context.
- Control panel: mode selection, return-to-home, and emergency actions.

## ROS2 Bridge Contract

Swarm Center communicates with ROS2 through `gcs_bridge`.

Selected incoming ROS2 state:

- `/swarm/task_status`
- `/swarm/drone_status`
- `/swarm/mission_ready`
- `/swarm/mission_complete`
- `/field/setup_status`
- `/field/setup_complete`

Selected outgoing operator intent:

- `/field/mission_confirm`
- `/field/generate_grid`
- `/swarm/mode`
- `/swarm/rth_request`
- `/swarm/manual_control`
- `/swarm/peer_cells`

The full topic contract is documented in `docs/topic_contract.md`.

## Phase 1-5 Milestone Scope

Validated in the public milestone:

- Multi-drone mission monitoring for the Gazebo `swarm_field` E2E workflow.
- Manual field setup through Swarm Center controls.
- Autonomous mapping mission monitoring.
- Camera and 3D map tabs.
- HTML report generation after mission completion.

Known non-blocking limitations are documented in
`docs/test_pahes1-5/known_issues.md`.
