================================================

Co overujeme:
- PHASE 1: obstacle_avoidance_runtime je jediny PX4 setpoint owner;
  swarm_agent bezi jako delegator (`navigation_backend=avoidance_runtime`);

TERMINAL 6 ŌĆö field_setup_tool (operator UI, curses)
---------------------------------------------------
```bash
cd /home/tj/_Data/_Projekty/TJlabs/scout_ws
source /opt/ros/jazzy/setup.bash
source install/setup.bash
ros2 run scout_control field_setup_tool
```

Klavesy:
- `1..N`  ŌĆö assign pad k aktualne ovladanemu dronu (pad_id)
- `M`     ŌĆö manual control mode toggle
- `C`     ŌĆö mark corner (LEGACY 4-corner mode, zpetna kompatibilita)
- `B`     ŌĆö mark boundary point (Phase 2B polygon vertex)
- `F`     ŌĆö finalize / close polygon (Phase 2B)
- `S`     ŌĆö start mission (`/field/mission_confirm`)
- `R`     ŌĆö RTH all
- `Q`     ŌĆö quit

DOPORUCENY POSTUP PRO PHASE 2 OVERENI:
1. Po startu drone_0 + drone_1 pockej, az field_setup_coordinator hlasi
   `ASSIGN_PADS`.
2. Stiskni `1` -> assign pad_0 dronu_0; `2` -> assign pad_1 dronu_1.
3. Coordinator prejde do `CAPTURE_BOUNDARY`.
4. Manualne preleti drone_0 po obvodu pole (manual mode `M`, WSAD).
5. Na kazdem rohu pole stiskni `B` (boundary point).
6. Po zachyceni 4-N polygon vertices stiskni `F` (close polygon).
7. Coordinator prejde do `GENERATE_GRID` -> `WAITING_FOR_LANDING`.
8. Po landed confirm prejde do `READY_FOR_MISSION`.
9. Stiskni `S` pro start mise.

PHASE 2 ACCEPTANCE CHECKS
-------------------------

[ ] **Pad metadata schema.**
```bash

import json
from pathlib import Path
d = json.loads(Path('/home/tj/_Data/_Projekty/TJlabs/scout_ws/perimeters/home_positions.json').read_text())
pads = d.get('pads', d if isinstance(d,list) else [])
required = {'pad_id','status','charging_capable','orientation_deg','service_priority'}
```

FULL E2E ACCEPTANCE CHECKS
--------------------------

[ ] **Mission start.**
Po stisku `S` ve `field_setup_tool` (nebo `mission_confirm` z swarm_center)
swarm_coordinator alokuje cely a posila `/{drone}/next_cell`.

[ ] **Cell delegation.**
swarm_agent forwarduje cell jako `/{drone}/avoidance/target_cmd`.
Sledovat:
```bash
ros2 topic echo /drone_0/avoidance/target_cmd
```

[ ] **CELL_COMPLETE z runtime feedbacku.**
Po dorazeni runtime nastavi `last_completed_target_id`. swarm_agent
emituje `CELL_COMPLETE` (`/swarm/task_status`).

[ ] **RTH all + landed confirm.**
RTH `R` -> oba drony se vrati na assigned pad. landed confirmation flipne
pad state.

[ ] **Zadne PX4 conflicty.**
Behem celeho behu `ros2 topic info /fmu/in/trajectory_setpoint -v` musi
porad hlasit prave 1 publisher per dron.
